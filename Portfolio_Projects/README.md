### 🏪 E-Commerce Enterprise Performance & Pipeline Integrity Audit

**Domain:** Business Intelligence, Data Engineering, and Revenue Operations  
**Dataset Scale:** 5,000-Row Normalized Transactional Ecosystem (`dbo.Orders` & `dbo.Products`)  
**Tech Stack:** Microsoft SQL Server (T-SQL) / SSMS

* * *

### 🎯 Executive Project Overview

This portfolio asset simulates an exhaustive **Commercial Operations & Data Integrity Audit** for a large-scale wholesale e-commerce platform. Rather than running isolated, superficial queries, this project maps out a multi-layered analytical engine designed to solve three high-priority executive business challenges:

1.  **Objective 1: Market Share & Concentration Analysis** – Segmenting purchasing behaviors into macro commercial cohorts to identify revenue exposure.
2.  **Objective 2: Ingestion Pipeline Quality Audit** – Systematically scanning historical transactional logs to expose data entry corruption trends over time.
3.  **Objective 3: Basket Density & Fulfillment Velocity Analysis** – Evaluating transaction slip distributions to optimize supply chain inventory packing protocols.

* * *

### 🛠️ Advanced Structural Architecture

To prove production-grade database mastery, the scripts inside this engine bypass brittle, flat query designs and leverage senior development paradigms:

*   **Stacked Common Table Expressions (CTEs):** Decoupling variable extraction layers from final aggregation arrays to guarantee query performance scales effortlessly.
*   **Non-Key Relational Bridging (`CROSS JOIN`):** Injecting isolated macro corporate benchmarks smoothly across row arrays to calculate market share distribution ratios on the fly.
*   **Defensive Datatype Engineering:** Mitigating standard silent execution traps, including **Integer Division truncation** and **Nullable Column data exclusion**.

* * *

### 🗂️ Core Repository Components

### 📁 File 1: Objective\_1\_Market\_Share\_Analysis

*   **Business Target:** Isolates wholesale customer volume tiers and calculates exactly what percentage share of global corporate revenue each segment commands.
*   **Core Logic:** Multiple reusable CTEs coupled with conditional `CASE WHEN` segmentation and a floating-point `CROSS JOIN` matrix.

***

### 📁 File 2: Objective\_2\_Pipeline\_Quality\_Audit

*   **Business Target:** Tracks historical system intake failure frequencies by calculating data corruption ratios across fiscal calendar timelines.
*   **Core Logic:** Chronological string extractions, binary null tracking flags, and precision decimal casting overrides (`100.0` numeric shift).

***

### 📁 File 3: Objective\_3\_Cart\_Velocity\_Analysis

*   **Business Target:** Measures fulfillment density by classifying checkout baskets by absolute item counts rather than raw unit components.
*   **Core Logic:** Containerized row consolidation pipelines leveraging nested aggregation routing and transaction slip frequency counting.

* * *

### 📈 Strategic Business Discoveries & Actionable Insights

*   **Wholesale Concentration Pattern:** The database profiles heavily as a specialized B2B wholesale operation. Orders display high financial values (exceeding 100k per transaction) with narrow product category variety, highlighting the need for corporate key-account retention playbooks.
*   **Fulfillment Optimization:** True transaction volume velocity tracks tightly to high-density single-item containers, indicating that supply chain operations should prioritize high-volume single-sku picking tracks over complex multi-item sorting setups.
